Common Credential and Region Parameters

Description

Most of the cmdlets in the AWS Tools for Windows PowerShell require that the user specify AWS access credentials and/or the AWS region that the operation should be performed in. To support these requirements, most cmdlets support the common set of parameters described below.

Credential Parameters

-AccessKey <String>
The AWS access key for the user account.
Required? False
Position? Named
Accept pipeline input? False
-SecretKey <String>
The AWS secret key for the user account.
Required? False
Position? Named
Accept pipeline input? False
-SessionToken <String>
Temporary session token obtained with Get-STSSessionToken.
Required? False
Position? Named
Accept pipeline input? False
-ProfileName <String>
The name associated with a set of credentials previously saved in either the encrypted SDK credential store (and shared with the AWS SDK for .NET and the AWS Toolkit for Visual Studio) or an ini-format credential file, shared with the SDKs and the AWS CLI.
Required? False
Position? Named
Accept pipeline input? False
-ProfileLocation <String>

Used to specify the name and location of the ini-format credential file (shared with the AWS CLI and other AWS SDKs) when the file does not use the default name and/or folder location.

When the ini-format credential file uses the default filename ('credentials') and is placed in the default search location ('C:\Users\userid\.aws') this parameter is not required. This parameter is also not required when the profile to be used is contained in the encrypted credential file shared with the AWS SDK for .NET and AWS Toolkit for Visual Studio.

As the current folder can vary in a shell or during script execution it is advised that you use specify a fully qualified path instead of a relative path.

Required? False
Position? Named
Accept pipeline input? False
-Credential <AWSCredentials>
An AWSCredentials object containing key information for the user account.
Required? False
Position? Named
Accept pipeline input? False

Region Parameters

-Region <String>
The system name of the AWS region in which the operation should be invoked. For example, us-east-1, eu-west-1 etc.
Required? False
Position? Named
Accept pipeline input? False

Supported Version
